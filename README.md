# 2026年高考志愿 · 家庭决策体检问卷

> 数据驱动 · 科学决策 · 家庭共识

一个面向高考家庭的 **志愿填报决策辅助工具**，通过多维度问卷帮助家长和学生发现认知差异、识别专业适配度，最终生成结构化的 JSON 数据供进一步分析使用。

---

## 项目概览

本项目是一个**纯前端单页应用**（SPA），使用原生 HTML / CSS / JavaScript 构建，无需任何构建工具或后端服务，可直接部署到任意静态服务器（如 GitHub Pages）。

### 技术栈

| 层 | 技术 |
|---|---|
| 页面结构 | HTML5（多视图 SPA） |
| 样式 | CSS3（移动优先、温暖学术风） |
| 应用逻辑 | 原生 JavaScript（ES5 兼容，全局作用域，供 HTML `onclick` 调用） |
| 图表 | [Chart.js 4.4](https://www.chartjs.org/)（雷达图） |
| 数据持久化 | `localStorage`（断点续填） |

---

## 核心功能

### 1. 双角色独立填写机制

问卷分为 **家长部分** 和 **学生部分**，且明确要求双方**独立填写、严禁商量**，以保证数据的真实性。

- **家长部分（8题）**：发展路径偏好、风险承受能力、面子与实惠、学历支撑上限、考公容忍度、意向专业方向、最看重特质、焦虑点
- **学生部分（14题）**：基本信息（性别/分数层次）、个人发展预期、学业能力与学科偏好、感官与性格底线评测、意向专业初筛

系统设计了一个 **家长→学生交接过渡页**（`view-student-transition`），在此处再次提醒学生独立填写。

### 2. 多维度专业适配度阻断模型（36题）

这是整份问卷的核心测评模块，基于 **霍兰德职业兴趣理论（RIASEC）** 设计：

| 维度 | 代码 | 含义 |
|---|---|---|
| 现实型 | R | 动手操作、务实执行 |
| 研究型 | I | 逻辑推理、深度钻研 |
| 艺术型 | A | 审美创意、情绪感知 |
| 社会型 | S | 人际互动、助人合作 |
| 企业型 | E | 目标导向、领导影响 |
| 常规型 | C | 条理规范、细节严谨 |

每题描述一种行为或偏好，用户选择「不符合 / 一般般 / 很符合」（对应 0/1/2 分），每个维度 6 题，满分 12 分。

#### 交互方式

- **手机端（触屏滑动）**：
  - 向右划 → 很符合
  - 向上划 → 一般般
  - 向左划 → 不符合
  - 卡片跟随手指移动，松手时飞出动画 + 下一题滑入
- **电脑端（按钮点击）**：底部三个按钮对应三种选择
- 支持**返回上一题**（`undo` 操作，会扣除对应分数）

### 3. 表单收集与验证

- **单选/多选/网格选择/自评表格/文本输入** 五种输入类型，统一由 `collectFormData()` 函数收集
- 提交时进行**逐题校验**，未完成的题目会弹出提示列表
- 第 9 题（感官耐受度）为选填，未选时自动设为 `null`

### 4. 数据导出

最终结果以**二进制文件** (`.bin`) 下载，并在结果页自动生成**数据二维码**供扫码获取。两种方式的数据内容完全一致。

解码方式：
```bash
# 从二进制文件解码
python decoder/decode_bin.py Input/高考志愿问卷结果_xxx.bin  Output/output.json

# 从二维码扫描文本解码（扫码后保存为 .txt 传入）
python decoder/decode_bin.py qr_scan.txt  Output/output.json

# 从截图/照片自动识别二维码
python decoder/scan_qr.py                # 扫描 Input/ → Output/
python decoder/scan_qr.py screenshot.png  # 扫描单张图片
```

解码后的结构化 JSON 包含三个部分：

```json
{
  "meta": { "version": "1.0", "date": "...", "title": "..." },
  "parent": { "q01_发展路径偏好": {...}, ... },
  "student": { "q01_性别": {...}, ... },
  "model36": {
    "name": "多维度专业适配度阻断模型",
    "scores": { "R": 8, "I": 10, "A": 4, "S": 6, "E": 5, "C": 7 },
    "answers": [{"id": 1, "type": "C", "score": 2}, ...]
  }
}
```

下载文件命名格式：`高考志愿问卷结果_2026-06-23T12-00-00.bin`（约 60–200 字节，vs 原 JSON 约 6KB，缩减约 98%）

### 5. 断点续填（localStorage 持久化）

每个阶段的数据都会自动保存到 `localStorage`：

- `qa_phase`：当前阶段（`parent` → `student` → `holland` → `done`）
- `qa_parent` / `qa_student` / `qa_holland`：各阶段原始数据

刷新页面后，首页会自动显示 **「继续填写」** 按钮，支持：
- 回到学生部分继续填写
- 回到 36 题测评（记住已答到第几题）
- 查看已完成的结果页

### 6. 结果二维码

完成问卷后，结果页自动生成一个 **数据二维码**，编码包含全部问卷结果。扫码即可传输数据，无需下载文件。

- **编码格式**：`GK1.` 前缀 + Base64 编码的二进制数据（约 110–200 字符）
- **生成库**：[qrcode-generator](https://github.com/kazuhikoarase/qrcode-generator)（CDN 加载，零本地依赖）
- **容错级别**：M（中等，可容忍约 15% 遮挡）

### 7. UI 设计

- **温暖学术色调**：深蓝 (`#1e3a5f`) 为主色，米白 (`#f4f1ec`) 为底色，搭配金色点缀
- **品牌标识**：顶部展示「会泽筑梦高考教育」Logo 和品牌名
- **信息层级清晰**：卡片式布局（`.card`）、警告框（`.warning-box`）、提示框（`.info-box`）
- **响应式适配**：移动端优先，`max-width: 520px` 居中布局，小屏优化字号和网格
- **粒子背景**：Canvas 实现的浮动粒子动画（默认隐藏，保留兼容）
- **视图切换动画**：`fadeIn` 淡入 + 上移动效
- **按钮按压反馈**：`:active` 状态 `scale(.97)` 缩放

1. 克隆仓库：
   ```bash
   git clone <repo-url>
   cd paid-questionnaire
   ```

## 设计原则

- **零依赖本地部署**：除 Chart.js 通过 CDN 加载外，无任何外部依赖
- **移动优先**：触屏滑动交互、响应式断点、大按钮设计
- **数据安全**：所有数据仅存储在用户浏览器的 `localStorage`，不上传至服务器
- **ES5 兼容**：JavaScript 使用 `var` 和传统函数声明，确保在旧浏览器中也能运行
- **容错处理**：图片加载失败自动隐藏（`onerror="this.style.display='none'"`）
